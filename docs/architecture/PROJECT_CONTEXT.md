## What this is

**Travel Operation** — a production-grade, SaaS-first operating system for travel
agencies, OTAs, visa agencies, B2B agents, ticketing companies, corporate travel
teams, and multi-branch travel businesses.

Non-negotiable properties: SaaS-first, tenant-safe, multi-branch, RBAC-controlled,
audit-first, API-first, Cloudflare-friendly, PostgreSQL-first, mobile-first,
security-first.

## Stack

| Layer     | Choice |
|-----------|--------|
| Frontend  | Next.js (App Router) + TypeScript + Tailwind + shadcn/ui + React Query |
| Backend   | NestJS + TypeScript, modular monolith, REST, Swagger |
| Database  | PostgreSQL (Neon in prod) + Prisma ORM |
| Storage   | Cloudflare R2 (private, signed URLs) |
| Edge      | Cloudflare Pages / DNS / CDN / WAF (frontend + edge only for now) |
| Package   | pnpm workspaces (pnpm ONLY — never npm/yarn) |

The NestJS API runs as a **separate Node service**. Cloudflare is NOT the backend
yet. PostgreSQL is the single source of truth. Workers/Queues/Workflows come later.

## Request/response contract (current)

- Success envelope: `{ success, data, timestamp }` (see TransformInterceptor).
  Target future shape adds `meta` + `traceId` — see `API_RULES.md`.
- List services return `{ data, total, page, limit, totalPages }`.
- `lib/api.ts` unwraps the outer `data`, so the frontend receives the inner payload.

## Naming conventions that already exist (IMPORTANT)

- **Permissions in code use `MODULE_ACTION`** (e.g. `LEAD_CREATE`, `CLIENT_READ`),
  generated by `@travelo/permissions` (`Modules` × `Actions`). The product spec asks
  for `module:action`; this is a **known divergence** tracked in `TENANT_RBAC_RULES.md`.
  Keep using the existing `MODULE_ACTION` form until a deliberate migration.
- Tenant scoping via `X-Tenant-Id` header, branch via `X-Branch-Id`.
- Soft delete via `deletedAt` on tenant business tables.
